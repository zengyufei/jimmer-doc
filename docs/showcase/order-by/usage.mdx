## Multiple orderBy actions

<Tabs groupId="language">
<TabItem value="java" label="Java">

```java
BookTable table = BookTable.$;

List<Book> books = sqlClient
    .createQuery(table)
    // highlight-next-line
    .orderBy(table.name().asc())
    // highlight-next-line
    .orderBy(table.edition().desc())
    .select(table)
    .execute();
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

```kotlin
val books = sqlClient
    .createQuery(Book::class) {
        // highlight-next-line
        orderBy(table.name.asc())
        // highlight-next-line
        orderBy(table.edition.desc())
        select(table)
    }
    .execute()
```

</TabItem>
</Tabs>

## orderBy with mutiple arguments

There is another way to write the equivalent of the above code

<Tabs groupId="language">
<TabItem value="java" label="Java">

```java
BookTable table = BookTable.$;

List<Book> books = sqlClient
    .createQuery(table)
    // highlight-next-line
    .orderBy(
        table.name().asc(), 
        table.edition().desc()
    )
    .select(table)
    .execute();
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

```kotlin
val books = sqlClient
    .createQuery(Book::class) {
        // highlight-next-line
        orderBy(
            table.name.asc(), 
            table.edition.desc()
        )
        select(table)
    }
    .execute()
```

</TabItem>
</Tabs>

However, the previous way of writing is more conducive to the organization of the code structure, so the previous way of writing is more recommended.

## Sorting based on subqueries

<Tabs groupId="language">
<TabItem value="java" label="Java">

```java
BookTable table = BookTable.$;
AuthorTableEx author = AuthorTableEx.$;

List<Book> books = sqlClient
    .createQuery(table)
    // highlight-next-line
    .orderBy(
        sqlClient
            .createSubQuery(author)
            .where(author.books().eq(table))
            .select(Expression.rowCount())
            .desc()
    )
    .select(table)
    .execute();
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">

